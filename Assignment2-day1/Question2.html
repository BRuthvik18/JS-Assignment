->Var is a variable where SCOPE of that variable is global,that can be accessed through out the code,it is also caleed as FUNCTION SCOPED

->if a variable is declared inside a function then it can be accessed only within the function

->Variables can be re declared and updated

->The problem with variable is ,it causes a lot of bugs

-> let is a blocked scope,So when a variable declared in a block,the variable is limited to be accssed in only that block

->Let can be UPDATED but not RE-DECLARED,If the variable is defined in different scopes there will be no error.

->Let is a better choice than var because it does not cause a problem.

->You don't have to worry about the variable which was declared as let in the previous block.

CONST ->The declared variables are to be intialized ,otherwise it shows a error.

->these variables are UNMUTABLE. ->Cannot be UPDATED or RE-DECLARED.
